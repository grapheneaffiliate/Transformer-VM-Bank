## Adding / removing maintainers

Per `GOVERNANCE.md`. Add via unanimous-consent ADR; remove via
majority-of-remaining ADR.

## Per-maintainer responsibilities

A maintainer commits to:
- Respond to PRs within ~5 business days (acknowledge, even if full
  review will take longer).
- Be reachable for security disclosures (see `SECURITY.md`).
- Participate in ADR reviews.
- Vote on release tags.

A maintainer is not committing to:
- Full-time work on PSL.
- 24/7 availability.
- Sole responsibility for any one component.
