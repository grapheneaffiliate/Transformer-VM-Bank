# PSL — Code of Conduct

## TL;DR

Be kind, be specific, be technical. Disagreements about code are
welcome. Personal attacks are not.

## Scope

This applies to:
- All interactions in this repository (issues, pull requests,
  discussions, code review).
- All interactions in PSL-affiliated communication channels (chat,
  mailing list, calls).

It does not police your behavior outside these spaces.

## Expected behavior

- Critique the work, not the author. "This implementation has a race
  condition because..." is great. "This implementation is sloppy" is
  lazy and unhelpful.
- Be specific. Vague criticism wastes everyone's time. If you think
  something is wrong, say what specifically and why.
- Assume good faith. Most disagreements come from missing context,
  not bad intent.
- Acknowledge uncertainty. "I think X but I haven't verified Y" is
  more honest and more useful than asserting confidently and then
  being wrong.

## Unacceptable behavior

- Personal attacks, slurs, harassment, threats. Includes via private
  channel; report it to maintainers if it happens off-list.
- Sustained disruption of discussion (concern-trolling, repeatedly
  re-litigating closed ADRs without new information).
- Sexualized language or imagery in any PSL space.
- Doxxing or sharing private information about contributors without
  their consent.

## Reporting

If you experience or witness behavior that violates this code:
- For routine issues, raise it in-channel ("hey, that comment was
  out of line"). Most issues resolve at this level.
- For serious issues, email the maintainers (see `MAINTAINERS.md`
  for current addresses) or file a private security report
  (`SECURITY.md` channel works for this too).

We aim to respond to reports within 7 days. Reports will be handled
confidentially; the reporter's identity will not be shared without
their explicit consent.

## Consequences

Maintainers may, at their discretion:
- Edit or remove inappropriate comments.
- Lock issues / pull requests temporarily.
- Ban a contributor from the repository (temporary or permanent) for
  serious or repeated violations.

For the rare case of a maintainer violating this code, the procedure
is the same — reports go to the other maintainers, who handle it
without the accused maintainer's involvement.
